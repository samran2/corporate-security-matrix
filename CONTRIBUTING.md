# Contributing

## Project direction

This repository is for a globally reusable corporate security matrix. Keep changes:

- English-first
- sector-neutral where possible
- practical for real operating teams
- backed by primary sources

## Source rules

When you add or change matrix content:

1. Use primary sources first.
2. Prefer standards bodies, intergovernmental organizations, official regulators and public framework owners.
3. Do not use vendor marketing, generic blogs or AI-generated citations as evidence.
4. Add or update the matching entry in `SOURCE-LIBRARY.md`.
5. Keep matrix wording original and concise.

## Content rules

- Keep threats concrete and operational.
- Keep controls actionable and implementation-oriented.
- Avoid country-specific legal claims unless clearly scoped.
- Prefer durable concepts over news-driven detail.
- Preserve the one-cell pattern: `Threats` and `Controls` together.

## Pull request checklist

- Matrix text stays in English.
- New content fits an existing domain or clearly justifies a new one.
- Source links are official and still live.
- No long copied text from standards or publications.
- The site still works as a no-build static page.
